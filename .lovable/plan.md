# Numerology Sword — Build Plan

A mystical dark-themed single-page app where a user enters their full name, date of birth, and gender, watches a 60-second ad/timer gate, and then receives a deep numerology "SWOT" reading generated by Lovable AI (Gemini).

## User Flow

1. **Landing screen** — hero with "Numerology Sword" branding, sword + sigil motif, short pitch, single CTA "Get Your Reading".
2. **Form screen** — three fields: Full Name (English, as on documents), Date of Birth (date picker), Gender (Male / Female / Other). Validation with zod.
3. **Ad gate screen** — Google AdSense banner displayed prominently, with a 60-second countdown ring. The "Reveal My Reading" button is disabled until the timer hits 0. While waiting, we kick off the AI generation in the background so the reading is ready (or near-ready) the moment the timer ends.
4. **Reading screen** — beautifully formatted markdown output of the full SWOT analysis with section headings, emojis, karmic-debt callouts, action plan. Buttons: "New Reading" and "Copy / Share".

## Style

Mystical dark theme:
- Background deep midnight (near-black with a hint of indigo)
- Primary accent: muted gold
- Secondary accent: arcane purple
- Subtle radial glow + faint rune/sword SVG motifs
- Serif display font for headings (e.g. Cinzel via Google Fonts), clean sans for body
- All colors as semantic oklch tokens in `src/styles.css` (no hard-coded colors in components)

## Tech

- **Frontend**: TanStack Start, React 19, Tailwind v4, shadcn/ui (Card, Button, Input, RadioGroup, Calendar/DatePicker), `react-markdown` for rendering the reading.
- **Backend**: Lovable Cloud enabled → Lovable AI Gateway. A `createServerFn` POST endpoint `generateReading` that takes `{ fullName, dob, gender }`, builds the prompt server-side (the user-supplied prompt above, with the data injected so the AI skips the question step and returns the analysis directly), and calls `google/gemini-2.5-pro` (best for deep reasoning + long output) non-streaming. Returns markdown.
- **AdSense**: Inject the AdSense script in `__root.tsx` head. Render an `<ins class="adsbygoogle">` slot on the gate screen. The user will need to provide their AdSense Publisher ID + Ad Slot ID after build (we'll add an input prompt for those, or hardcode placeholders they can swap).

## Routes

- `/` — single-page experience with internal step state (landing → form → gate → reading). Keeps things simple and avoids losing form state on refresh. SEO meta on root: title "Numerology Sword — AI Numerology Reading", description, og tags.

## Honest Caveats (for the user)

- **AdSense isn't truly "non-skippable"** — it's a banner ad plus a forced 60-second wait timer on our side. Forcing users to wait on ads can violate AdSense policies; if your account gets flagged, switch to a paid unlock or remove the forced wait. This is the closest compliant approximation.
- **AdSense Publisher ID + Ad Slot ID** are required from you after the app is built — I'll leave clearly marked placeholders.
- Approval for AdSense takes time and requires a published, content-rich site; the ad slot will show blank until your account is approved.
